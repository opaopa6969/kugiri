# 分岐エントロピー（Branching Entropy）

[English version](branching-entropy.md)

---

## これは何？

**分岐エントロピー（branching entropy）** は、「ある文字列のすぐ次（または前）に、
どれだけ **いろんな文字** が来るか」を測る数値です。

道路にたとえると分かりやすいです。一本道なら次に進む先は 1 つだけ。でも交差点に来ると、
次に進める方向が一気に増えます。**「次に来る文字の種類が急に増える場所」＝交差点＝言葉の切れ目**、
というのが分岐エントロピーの発想です。語の途中では次の文字はほぼ決まっている（エントロピー低）、
語の終わりでは次に何が来るか自由（エントロピー高）になります。

---

## なぜ重要なのか？

kugiri の主目的は、正解ラベルの無い [字](aza.ja.md) を教師なしで切り出すことです。
「ここが切れ目だ」という教師がいないので、**切れ目を文字列の統計から見つける**必要があります。

分岐エントロピーは、その「切れ目らしさ」を測る主力の信号です。
字の名前がコーパス中で繰り返し現れると、その境界の外側にはいろんな文字が続く
（＝高エントロピー）。この性質を使って、字彙（字の語彙）の候補を選び出します。

---

## どう動くのか？

ある部分文字列について、右隣（または左隣）に来た文字の分布を数え、その散らばりを
[log](../study/05-log-to-jouhou.md) を使ったエントロピーで測ります。
散らばっている（多様）ほど高く、偏っている（いつも同じ字）ほど低くなります。

```
  "中里" の右隣に来た文字を数える：
    中里[1] 中里[番] 中里[、] 中里[前] …  → バラバラ＝高エントロピー＝切れ目っぽい
  "中" の右隣：
    中[里] 中[里] 中[里] …               → いつも里＝低エントロピー＝語の途中

  ※ 文字列の端（先頭・末尾）は境界トークン BND として扱う
```

`aza/AzaInducer` では、候補を字彙に **採用（admit）** する条件としてこれを使います。

```
admit: 頻度 cnt >= minCount
       かつ 左右の隣接が多様（>=2種 または端）
       かつ 平均分岐エントロピー >= eMin
```

ロードマップ T4 では、この境界スコアを [PMI](pmi.ja.md) 剪定と **明示的に併用** して
内部境界の再現率を上げる予定です。

---

## kugiri ではどう使われている？

実装は `aza/AzaInducer` の字彙誘導（fit）段で、頻度・左右隣接の多様性とあわせて
分岐エントロピーを採用条件にしています。採用された字彙は最終的に
[Viterbi](viterbi.ja.md) によるユニグラム最尤分割で字の切れ目に変換されます。
