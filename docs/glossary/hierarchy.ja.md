# 階層要素（hierarchy）

[English version](hierarchy.md)

---

## これは何？

階層要素（hierarchy）は、日本の住所を構成する **部品の種類** の一覧です。
都道府県 / 市 / 区 / 群 / 町または大字 / 丁目 / 字小字 / 地番 / 街区符号 /
住居番号 / 枝番号 / 棟 / 部屋番号 / 方書き…と、大きいものから小さいものへ階層をなします。

住所を「だんだん細かくしていく入れ子の箱」だと思ってください。
いちばん外の箱が都道府県、その中に市、その中に区や町、さらに丁目や番地……。
kugiri がやりたいのは、1本につながった住所文字列を、この箱（種類）ごとに切り分けることです。

---

## なぜ重要なのか？

切り出した部品に「どの種類か」のラベルが付いていないと、住所の意味的な処理ができません。
`千代田区` が区なのか市なのか、`丸の内` が町なのか字なのかが決まって初めて、
名寄せや照合・正規化が成り立ちます。

そして種類には2つの顔があります。文字列として実体を持つもの（都道府県・丁目・地番…）と、
持たないもの（「国域のトップ」「町域のトップ」のような抽象ノード）。
この区別を最初に決めておかないと、**実体の無いノードに無理やりラベルを当てて**しまい、
出力ラベルの集合が壊れます。

---

## どう動くのか？

階層要素には2種類あります。

```
構造マーカー (isTop=true)        表層リーフ (isTop=false)
─────────────────────────       ─────────────────────────
国域Top / 町域Top / 建物Bottom   都道府県 / 市 / 区 / 丁目
など、抽象ノード                 字小字 / 地番 / 街区符号 など
文字列スパンを持たない           実際に部分文字列に対応する
level から導けるので             これだけを系列ラベルにする
出力ラベルから外す               → [BIOES] のタグに使う
```

出力に使うのは「表層リーフ」だけ。構造マーカーは level（深さ）から導出できるので、
ラベルからは外します。表層リーフ → level の写像は `label/Labels.java` が持ちます。

---

## kugiri ではどう使われている？

`label/Hierarchy.java` が原 `階層要素` enum を忠実に移植したもので、各要素の level と
isTop（構造マーカーか表層リーフか）を保持します。出力ラベルの文字列はこの enum と **1:1 対応**
を崩しません。`label/Labels.java` が表層リーフ → level の写像（出力ラベル集合の定義）を担い、
[BIOES](bioes.ja.md) のタグ空間がこの上に作られます。
ラベルの付かない `字小字`（[aza](aza.ja.md)）は、[ABR](abr.ja.md) の頭ラベルと残差スロットから
教師なしで推定します。
