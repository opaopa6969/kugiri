# 字（字小字 / 教師なし字推定）

[English version](aza.md)

---

## これは何？

**字（あざ）／小字（こあざ）** は、日本の住所に古くから残る「土地の細かい呼び名」です。
たとえば `岩手県盛岡市上田字小鳥沢` の `小鳥沢` の部分が字です。
都道府県・市・区・町・丁目・番地…といった他の階層要素には「誰かが付けてくれたラベル」が
あるのに対し、字は **公的なラベル付きデータ（ABR / KEN_ALL）でも一部しか整備されていません**。

kugiri の主目的は、この **ラベルの無い字を、教師（正解ラベル）ゼロで推定する** ことです。
教師あり学習が頼る「正解」が存在しない領域を、住所コーパス全体の統計だけで切り出します。
学習コースでは、ラベルをもらえず姿が見えなくなった精霊「アザミ」を探す物語として描かれています。

---

## なぜ重要なのか？

住所を [階層要素](hierarchy.ja.md) ごとに切り出すとき、頭（都道府県〜大字）と
末尾（地番・番地）は外部辞書や正規表現で機械的に決まります。問題はその **間に残るスパン** です。

```
  岩手県盛岡市上田 小鳥沢 1234番地
  └─ABRで決まる頭─┘ └ ? ┘ └数字塊=地番─┘
                    ↑
              ここに残るのが「字」
              でも正解ラベルは誰も持っていない
```

ここを正しく切れないと、住所の機械処理（名寄せ・照合・正規化）が末尾でつまずきます。
しかも字は数が膨大で手作業のラベル付けは現実的でないため、**教師なしで当てる**ことに価値があります。

---

## どう動くのか？

kugiri は字を「3つの構造的な根拠」から推定します（詳しくは
[第12章 残差スロット](../study/12-zansa-slot.md) 〜 [第16章 全工程](../study/16-aza-zentai.md)）。

### 1. 残差スロット（消去法）

[ABR](abr.ja.md) が大字を、末尾の数字塊が [地番](hierarchy.ja.md) の開始を決めます。
その **間に残る非空スパンは、誰もラベルしなくても構造的に字** です（`Aza.peel()` が切り出す）。

### 2. 反復性（教師なし字彙誘導）

字名は全国の住所で繰り返し現れます。残差をコーパス横断で集め、
**頻出 × 左右に多様な隣接（[分岐エントロピー](branching-entropy.ja.md) が高い）** 部分文字列を
**字彙（字の語彙）として誘導** します。

### 3. PMI で癒着を剪定

`中里前田` のように、たまたま連接しただけ（[PMI](pmi.ja.md) が低い）の語は合成語として
字彙から除去します。`田中` のように常に共起（PMI 高）するものは1単位として保持。
残った字彙でユニグラム言語モデルの最尤分割（[Viterbi](viterbi.ja.md)）を行い、字の切れ目を決めます。

```
fit(residuals):
  全部分文字列の頻度 cnt と左右隣接分布を集計
  admit: cnt>=minCount かつ 左右が多様 かつ 平均分岐エントロピー>=eMin
  prune: PMI が tau 以下の連接を合成語として削除
segment(name):
  ユニグラム logp − wordPenalty を最大化する Viterbi
```

ABR/KEN_ALL は「大字辞書＋数字境界」という足場を与えるだけで、**字のラベルは1つも要りません**。

---

## kugiri ではどう使われている？

公開窓口は `aza/Aza`（残差スロット切り出し）と `aza/AzaInducer`（字彙誘導＋PMI 剪定＋最尤分割）です。
`demo/AzaDemo` が、字を隠した合成データで end-to-end を実演します。

`AzaDemo` の実測（**教師ラベル完全ゼロ**）:

```
内部境界(字の切れ目) P=1.000 R=0.708 F1=0.829
字スパン完全一致      353/400 = 0.883
```

精度 1.000（誤った切れ目を入れない）、再現は閾値（`tau` / `wordPenalty` / `minCount`）依存です。
実データは数千万件あり字の反復が桁違いに効くので、玩具コーパスより遥かに頑健になります。
