# codepoint（コードポイント）

[English version](codepoint.md)

---

## これは何？

codepoint（コードポイント）は、Unicode が1つ1つの文字に割り当てた **番号** です。
`A` は U+0041、`あ` は U+3042、`𠮷`（つちよし）は U+20BB7。
「人間が1文字だと思うもの」に、ちょうど1つの番号が対応します。

ここで気をつけたいのが、プログラミング言語の「char（文字型）」とは別物だということです。
Java の `char` は16ビットしかないので、`𠮷` のような大きな番号の文字は
**2つの char に割れて**（サロゲートペア）格納されます。
codepoint は割れません。`𠮷` はあくまで1 codepoint。住所を扱うときは、この「割れない単位」で
数えることがとても大切になります。

---

## なぜ重要なのか？

住所には外字（がいじ）や珍しい漢字がよく出てきます。
`𠮷田` さんの `𠮷`、地名の難読漢字、企業の私用領域（PUA）文字……。

もし `char` 単位でラベルを付けると、`𠮷` は2つに割れ、片方だけに `B-`、片方に `I-` のような
**意味のない境界が中に入ってしまう** か、添字がずれて位置を見失います。
codepoint 単位で扱えば、`𠮷` は最初から最後まで1つ。
[BIOES](bioes.ja.md) の旗も1本だけ立ち、位置のずれが起きません。

---

## どう動くのか？

文字列を `char` の列ではなく、codepoint の列に変換してから処理します。
Java では `String.codePoints()` が標準で用意されています。

```
"𠮷田" を char で見ると（長さ3）:
  [\uD842, \uDFB7, '田']   ← 𠮷 が2つに割れている！

codepoint で見ると（長さ2）:
  [0x20BB7, 0x7530]        ← 𠮷 は1つ、田 は1つ
   └─ 𠮷 ─┘ └─ 田 ─┘

ラベルもこの2要素に1つずつ対応する:
  [S-名前, ...]
```

外字（PUA）も大きな番号の漢字も、すべて「1 codepoint = 1要素」として素直に流れます。

---

## kugiri ではどう使われている？

`label/CodePoints.java`（`CodePoints.of(...)`）が、文字列と codepoint 単位列の相互変換を担当します。
kugiri は **codepoint 単位の系列ラベリング** を基本方針とし、`char` では扱いません。
これは unlaxer の CodePoint 基盤と一致し、サロゲートペアや外字（PUA）が1単位で流れるので、
名寄せの gaiji（外字）変換テーブルとも素直に突き合わせられます。
[BIOES](bioes.ja.md) のタグも [階層要素](hierarchy.ja.md) のスパンも、この codepoint 列の上に乗ります。
