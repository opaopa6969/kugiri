# パーセプトロンタガー（平均化構造化パーセプトロン）

[English version](perceptron-tagger.md)

---

## これは何？

**パーセプトロンタガー** は、住所の文字列を 1 文字ずつ見て「この文字は都道府県？ 市？ 字？」と
ラベルを貼っていく系列ラベラ（sequence labeler）です。kugiri では
**平均化構造化パーセプトロン（averaged structured perceptron）＋ [Viterbi](viterbi.ja.md)** で実装されています。

たとえるなら、経験を積んだ郵便局の仕分け係です。最初は何も知らず当てずっぽうで仕分けしますが、
正解を見せられるたびに「ここを間違えた、次はこう直そう」と手元のルール（重み）を少しずつ修正します。
何千通も練習すると、初見の住所もかなり正確に仕分けられるようになります。

線形 CRF（条件付き確率場）と同じ族の手法ですが、**外部ライブラリゼロの純 JDK** で動く点が特徴です。

---

## なぜ重要なのか？

日本の住所には分かち書きがありません。`東京都千代田区丸の内一丁目` を機械が部品に分けるには、
**どこからどこまでが 1 つの [階層要素](hierarchy.ja.md) か**を文字単位で判定する必要があります。

固定ルール（正規表現）でも一部は切れますが、表記ゆれや未知の地名にすぐ破綻します。
学習する系列ラベラなら、**手がかり（素性）から境界を一般化** できるので、未知の住所にも強くなります。

kugiri はまず「依存ゼロで動く参照実装」を欲しがりました。重い ML ライブラリを本体に持ち込まず、
それでも CRF と同等の構造（系列全体を見て最良のラベル列を選ぶ）を持つ —— その答えがこのタガーです。

---

## どう動くのか？

1. 各文字を **素性ベクトル**（手がかりの集合）に変換します。
   素性は `feature/Features.java` 由来で、**窓 ±2 の文字・文字種・接尾辞キーワード・bigram** など。
2. 重みと素性の内積で各ラベルのスコアを出し、**[Viterbi](viterbi.ja.md)** で系列全体の最良ラベル列を選びます。
   このとき [BIOES](bioes.ja.md) の不正な遷移（`I-` が単独で始まる等）はマスクして排除します。
3. 予測が正解とずれたら、**正解側の素性を加点・誤り側を減点**（パーセプトロン更新）。
4. 全エポックの重みを **平均（averaging）** して使う。これが過学習を抑え、汎化を効かせます。

```
  文字列:   東 京 都 千 代 田 区 ...
            │  │  │  │  │  │  │
  素性化:  [窓±2の文字 / 文字種 / 接尾辞KW / bigram]
            │  │  │  │  │  │  │
  スコア:  各ラベルへの内積  → Viterbiで系列全体を最適化（BIOES合法のみ）
            ↓
  予測:    B-都道府県 I-都道府県 E-都道府県 B-市 ...
            ↓ 正解と比較
  更新:    正解素性 +1 / 誤り素性 −1   →  最後に全重みを平均
```

---

## kugiri ではどう使われている？

本体は `tagger/PerceptronTagger.java`（系列ラベラ・Viterbi デコード）です。
素性抽出は `feature/Features.java`。学習・parse・評価の公開窓口は `tagger/AddressParser.java`。

CRF ではなくパーセプトロンを選んだのは、**依存ゼロの純 JDK で動く参照実装**にするためです。
精度が頭打ちになったら、同じ素性関数のまま **MALLET の CRF** や **文字 BERT ＋ ONNX** へ
差し替えられる設計になっています（タガーは差し替え可能なスロット）。
