# 弱教師（弱教師あり学習 / Weak Supervision）

[English version](weak-supervision.md)

---

## これは何？

弱教師（Weak Supervision）とは、**人手で1つ1つ正解ラベルを付ける代わりに、
外部の構造化データから「だいたい正しい・安いラベル」を自動生成する**やり方です。

たとえるなら、全部の答案を先生が手で採点するのではなく、
**模範解答集（既存の住所データベース）を機械で突き合わせて、ざっくり丸を付けてしまう**ようなもの。
完璧ではない（ノイズが混じる）けれど、**ほぼタダで大量に作れる**のが強みです。

kugiri では、住所の**頭の部分**（都道府県・市・区・群・町・丁目・[街区符号](hierarchy.ja.md)・住居番号）の
ラベルを [KEN_ALL](ken-all.ja.md) と [ABR](abr.ja.md) から自動で付けます。

---

## なぜ重要なのか？

[字（aza）](aza.ja.md) を教師なしで推定するには、その**足場**が要ります。
住所の頭がどこまでで、末尾の数字塊（地番）がどこから始まるかが分かって初めて、
「間に残るスパン＝字」という消去法が成立するからです。

弱教師は、この足場を**人手ゼロ**で用意します。

```
  住所:  岩手県盛岡市上田 小鳥沢 1234番地

  弱教師（KEN_ALL + ABR）でラベル化できる：
    岩手県=都道府県 / 盛岡市=市 / 上田=大字 ... 1234番地=地番
                          ↑       ↑
                       ここまで頭  ここから尻尾
                              \   /
                            残るのが字（aza）= 教師なしで推定
```

ただし「弱い」理由は、**頭しかカバーできない**点にあります。
字を含む尻尾には正解が1つも無い ── そこは [合成データ](synthetic-data.ja.md) のような
完全教師とは対照的に、教師なし推定に委ねられます。

---

## どう動くのか？

KEN_ALL は ZIP→(都道府県, 市区町村, 町域) を引けますが団体コードしか持たず、
ABR は町字・街区・住居まで分解済みだが ZIP を持ちません。そこで **opa 方式**で橋渡しします。

```
  ① KEN_ALL の ZIP で大まかに町域を引く
        020-0021 → 岩手県 / 盛岡市 / 上田
                       │
  ② その町域を ABR で引き直して頭を細かく分離
        盛岡市 → 市:盛岡市
        千代田区(特別区) → 東京23区:千代田区
        横浜市西区(政令市) → 市:横浜市 ＋ 区:西区
        虻田郡洞爺湖町(郡)  → 群:虻田郡 ＋ 町村:洞爺湖町
                       │
  ③ 突合は「文字列」で行う（団体コードは KEN_ALL 5桁 / ABR 6桁 で差があるため）
```

連結アライメントの妙で、コンポーネント列 `[(ラベル, 表層), …]` を連結すれば
codepoint 単位ラベルが**アライメント計算なしで**確定します。

---

## kugiri ではどう使われている？

頭ラベルの生成は `abr/Abr.java`（KEN_ALL→ABR 再解決 → Component 列）が窓口で、
`abr/KenAll.java` と `abr/Csv.java` が下支えします。`demo/AbrDemo` が
東京23区 / 市+区 / 群+町村 の分離を実演します（未マッチ率もログ出力）。

こうして作った頭ラベルが、`aza/AzaInducer` のゼロラベル字推定の**種**になります。
self-training（半教師）では、この頭ラベル＋字推定を種に、高信頼予測のみ擬似ラベル化して
再学習を反復する正攻法も用意されています。
