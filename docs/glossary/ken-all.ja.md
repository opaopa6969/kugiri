# KEN_ALL（日本郵便の郵便番号データ）

[English version](ken-all.md)

---

## これは何？

**KEN_ALL.CSV** は、日本郵便が配布する郵便番号データです。
郵便番号（ZIP）1 件ごとに、`都道府県 / 市区町村 / 町域` が並んだ巨大な表だと思ってください。

電話帳の逆引きのようなものです。番号を知っていれば、対応する「だいたいの住所」を
すぐ引ける。kugiri にとっては、ZIP を鍵にして住所の **頭（lv1–5：都道府県〜町域）**
を取り出すための、いちばん手軽な入口です。

---

## なぜ重要なのか？

[ABR](abr.ja.md) は住所を項目ごとに細かく割ってくれますが、**ZIP を持っていません**。
一方 KEN_ALL は ZIP を持っています。そこで kugiri は、

1. **KEN_ALL** で ZIP から「だいたいの住所（都道府県・市区町村・町域）」を引き、
2. その町域を **ABR** で引き直して、項目ごとに割れたラベルを得る、

という二段構えを取ります。KEN_ALL はその **第一段＝足場** を担います。
これが [弱教師](weak-supervision.ja.md) ラベル生成の出発点です。

---

## どう動くのか？

KEN_ALL から引いた頭の文字列を、住所文字列の **先頭への最長一致** で当てはめて、
[階層要素](hierarchy.ja.md) のラベルを付けます。

```
  住所:  岩手県盛岡市上田1234番地
  ZIP:   020-0021
          │ KEN_ALL を引く
          ▼
  都道府県=岩手県  市区町村=盛岡市  町域=上田
          │ 先頭への最長一致でラベル付け
          ▼
  [都道府県:岩手県][市:盛岡市][町または大字:上田] 1234番地…
```

注意点が 2 つあります。

- **文字コードは CP932（Windows-31J）**。UTF-8 で読むと文字化けします
  （リポジトリのサンプルだけは UTF-8）。
- **市区町村が連結されている**ことがある（例：`横浜市西区`）。これは [ABR](abr.ja.md) で
  さらに `市:横浜市 + 区:西区` のように割ります。

京都の通り名や岩手の地割など特殊な町域表記は `cleanTown` で個別に整えます。

---

## kugiri ではどう使われている？

実装は `abr/KenAll.java` で、`KEN_ALL.CSV → (zip7, 都道府県, 市区町村, 町域)` を提供します。
ここで得た頭ラベルを [ABR](abr.ja.md) の再解決へ渡し、`synth/Synth.fromRecords` 経由で
学習データに変換します。`demo/AbrDemo` で一連の流れを確認できます。

> 実 KEN_ALL の生データはコミットしません（サンプルのみ）。実データは CP932 で読み込みます。
