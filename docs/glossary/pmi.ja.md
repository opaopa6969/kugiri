# PMI（自己相互情報量）

[English version](pmi.md)

---

## これは何？

PMI（Pointwise Mutual Information / 自己相互情報量）は、「2つのものが**たまたま隣にいる**のか、
それとも**いつも一緒に出てくる仲間**なのか」を1つの数字で表す指標です。

```
PMI(a, b) = log( c(ab) * N / ( c(a) * c(b) ) )
```

たとえるなら、学校の休み時間に2人の生徒がいつも一緒にいるのを見て、
「この2人は本当の親友なのか、それともたまたま同じ場所に居合わせただけなのか」を
出席記録から見抜くようなものです。

- **PMI が高い** → `田中` のように、a と b が偶然では説明できないほど一緒に出る = **1つのまとまり**。
- **PMI が低い** → `中里前田` のように、たまたま隣り合っただけ = **本当は別々の2語**。

---

## なぜ重要なのか？

[字（aza）](aza.ja.md) を教師なしで切り出すとき、「残差スロット」には
複数の字がくっついて並ぶことがあります。`中里前田` は本当は `中里` ＋ `前田` という2つの字なのに、
頻度や [分岐エントロピー](branching-entropy.ja.md) だけ見ると「よく出る1語」に見えてしまいます。

ここで頻度だけを頼りにすると、**くっついた合成語をそのまま1つの字として誤登録**してしまいます。
PMI は「本物のまとまり（田中）」と「たまたまの連接（中里前田）」を区別する**判別信号**として、
この誤りを防ぎます。

---

## どう動くのか？

`AzaInducer` は、語彙候補 `ab` を既知の2分割 `a + b` に分けたとき、その PMI を計算します。

```
        c(ab) * N
PMI = log ─────────────
        c(a) * c(b)

  c(ab) … "ab" が現れた回数
  c(a)  … "a"  が現れた回数
  c(b)  … "b"  が現れた回数
  N     … 残差レコードの総数   ★ここが超重要

PMI が低い（<= tau）
  → a と b は独立にたまたま隣り合っただけ
  → "ab" は合成語なので字彙から削除（prune）

  例: 中里前田  →  PMI 低い  →  削除 → 中里 / 前田 に割れる
      田中      →  PMI 高い  →  保持 → 1語のまま
```

**正規化の落とし穴（DESIGN 付録で踏んだバグ）**:
PMI の `N` は **残差レコードの件数** で正規化しなければなりません。
全部分文字列の頻度和 `Z` で割ってしまうと、PMI が一様に過大評価され、
**剪定がまったく効かなくなります**。`N` か `Z` か、ここを取り違えると静かに壊れます。

---

## kugiri ではどう使われている？

PMI 剪定は `aza/AzaInducer` の `fit()` の中で行われます。
頻度と分岐エントロピーで語彙候補を集めた後（admit）、PMI が `tau` 以下の連接を
合成語として除去（prune）し、残った字彙でユニグラム言語モデルの最尤分割
（[Viterbi](viterbi.ja.md)）を行います。

`tau` は閾値で、実データの hold-out で再調整します。
DESIGN の設計判断ログでは、「頻度×長さの最大化 → 過分割」「単語ペナルティを上げる → 癒着」という
2つの失敗を経て、**判別信号は PMI である**と結論づけています。
