# kugiri 学習コース 第二部 〜姿(スガタ)は、ひとり？それとも？〜

> 第一部（[`../study/`](../study/README.md)）は、ラベルが無くて姿の見えない「字」の精霊
> **アザミ** を、教師なしで見つけ出す物語でした。
> 第二部は、その逆——**いくつもの姿（表記ゆれ）で現れる**精霊 **スガタ** の物語です。

## 🧚 第二部のあらすじ（テーマ）

同じ住所に、よく似た建物名がいくつも並ぶことがあります。

```
白雲荘 / 青雲荘
ライオンズマンション梅田 / ライオンズ梅田
コーポ山田 / コ−ポ山田
第一宿舎 / 第二宿舎
```

精霊 **スガタ** は、こうやって**たくさんの姿**で現れます。問いはいつも一つ：

> **その姿たちは「ひとり（同じ建物の表記ゆれ）」なのか、「別人（本当に別の建物）」なのか？**

文字の見た目（編集距離）だけでは解けません。`白雲荘`/`青雲荘` は1文字違いbut別建物、
`ライオンズマンション梅田`/`ライオンズ梅田` は5文字違いbut同じ建物。
**「違っている部分が“意味のある対立”か、ただの“notation/種別”か」**——これを
第一部で学んだ **分岐エントロピー・PMI・誘導語彙** で見抜くのが第二部です。

さらに第二部では、建物名を切り出し（境界＝分岐エントロピー＋系列ラベラ）、
行を **建物→棟→階→部屋** の木に組み、**永続化**して検索できるようにします。
状態を持たない第一部とは毛色が違う、「**覚えて・貯めて・引く**」世界へ。

## 🧩 「複数」の本当の難しさ（ドメイン類型）

「同じ住所に複数の名前」は意味が一様でない（詳細は [`../building/DESIGN.md`](../building/DESIGN.md) §0.1）：

- **A 複数施設の敷地**（自衛隊基地の宿舎/コンビニ/○○棟、団地）→ 別建物が複数（敷地の子）
- **B 本当に2棟** → 別建物
- **C 改名** → 同一（時系列エイリアス）
- **D notation 揺れ**（コーポ山田/コ−ポ山田）→ 同一
- **E 正式名 vs 略称**（県立○○施設○○寮 ⊇ ○○寮）→ 同一（**包含**）
- **F 略しすぎ衝突**（どちらも「寮」だけ）→ **テキスト不能 → 要レビュー**＋住所/部屋証拠

スガタの正体はこの **A〜F の見分け**。判定は **SAME / DISTINCT / NEEDS_REVIEW** の3値。

## ✅ Phase0（実証ずみ）

同一性プロトタイプ（`building/` モジュール `identity/`）で 9 ペア（D/E/B/A/typo/棟差/F）を
判定し、編集距離ベースラインと対決：

| 方式 | 正答 |
|---|---|
| **kugiri 統計（包含＋対立度）** | **9/9** |
| 編集距離（正規化, BH現行相当） | 5/9 |
| 編集距離（絶対≤4, BH legacy相当） | 3/9 |

編集距離は **NEEDS_REVIEW を出せない**ので型F（衝突）で必ず外す。kugiri は
`県立さくら施設さくら寮/さくら寮＝同一（略称）`・`ライオンズ梅田/ライオンズ難波＝別（地名）`・
`寮/さくら寮＝要レビュー` を正答。

実行：`mvn -q -f building/pom.xml exec:java -Dexec.mainClass=org.unlaxer.kugiri.building.demo.IdentityProbeDemo -Dstdout.encoding=UTF-8`

## 📚 章立て（Phase1 で本文執筆）

1. プロローグ：同じ顔、ちがう名前（スガタ登場）
2. 文字列距離だけでは解けない（白雲荘/青雲荘 の罠）
3. 種別語と固有名核（PMI で“ありふれた語”を消す）
4. 対立度：その違いは意味があるか、ノイズか（分岐エントロピー再訪）
5. クラスタリングのきほん（編集距離・連結法・しきい値）
6. 建物名を切り出す（境界＝分岐エントロピー、系列ラベラ再訪）
7. 行を木に組む（建物→棟→階→部屋）
8. 永続化と検索（なぜ状態が要るか）
9. アルゴリズムを切り替える（差し替え層・対決ハーネス）
