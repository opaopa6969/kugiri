# kugiri — 設計ドキュメント

> 機械学習による日本住所の階層トークナイザ生成器。
> ZIP＋住所文字列を `階層要素` ごとの token 列へ切り出す。
> **ラベルの無い「字(小字)」を教師なしで推定する**ことが主眼。

プロジェクト名 **kugiri（区切り）**: 階層トークナイザの本質「区切る」をそのまま表す。
groupId は unlaxer アンブレラに合わせ `org.unlaxer`。

## 1. 要件

会話を通じて固まった要件:

- **入力**: 郵便番号（ZIP）＋ 住所文字列。全国分を保有している前提。
- **出力**: 住所を `階層要素`（都道府県 / 市 / 区 / 群 / 町または大字 / 丁目 / 字小字 /
  地番 / 街区符号 / 住居番号 / 枝番号 / 棟 / 部屋番号 / 方書き …）ごとの token に切り出す。
- **方式**: 機械学習で tokenize する「parser ジェネレータ」。固定ルールではなく学習で汎化。
- **核心的制約**: 与えられているのは多くの場合 **未分割の生文字列**。階層境界の正解は無い。
- **真の主目的**: 日本の住所には **字(小字)** 等があり、それは ABR でも一部しか入っていない。
  **教師情報が無いところから、この階層構造を推定する**のが ML にやらせたいこと（＝みそ）。

## 2. 問題の定式化

### 2.1 codepoint 単位の系列ラベリング（char-level NER）

日本語住所は分かち書きが無いので、**1 codepoint ごとに所属する `階層要素` をラベル付け**する。
BIOES（=BILOU）で `B-丁目 / I-丁目 / E-丁目 / S-地番 / O` のように展開する。

codepoint 単位であることが重要:
- unlaxer の CodePoint 基盤と一致（Java では `String.codePoints()`、`CodePoints.of()`）。
- サロゲートペア（𠮷 等）も **外字(PUA)** も 1 単位として素直に流れる → 名寄せの gaiji 変換と整合。

### 2.2 階層要素 → 表層ラベルの写像

`階層要素` enum には2種類ある:
- **構造マーカー**（国域Top1-4 / 町域Top1-4 / 建物BottomN, `isTop=true`）:
  文字列スパンを持たない抽象ノード。level から導出できるので **出力ラベルから外す**。
- **表層リーフ**（都道府県 / 市 / 丁目 / 字小字 / 地番 …, `isTop=false`）:
  実際に部分文字列に対応する。**これだけを系列ラベルにする**。

実装: `label/Hierarchy.java`（原 enum の忠実移植・参照用）、`label/Labels.java`
（表層ラベル → level の写像）、`label/Bioes.java`（BIOES エンコード/デコード）。

## 3. データ戦略（KEN_ALL + ABR の弱教師）

### 3.1 連結アライメントの妙（学習データ生成の核）

住所を「コンポーネント列」 `[(ラベル, 表層), …]` として持てば、連結して文字列を作ると同時に
codepoint 単位ラベルが **アライメント計算なしで** 確定する。

```
[(都道府県,東京都),(東京23区,千代田区),(町または大字,丸の内),(丁目,一丁目),(街区符号,9番),(住居番号,1号)]
 → "東京都千代田区丸の内一丁目9番1号" ＋ 連結位置から BIOES が自動確定
```

実装: `synth/Synth.buildExample()` / `fromRecords()`。

### 3.2 KEN_ALL → ABR で頭をラベル化（実データ）

| 層 | ソース | 引き方 |
|----|--------|--------|
| 都道府県〜市区町村〜町域 (lv1–5) | **KEN_ALL.CSV**（日本郵便） | ZIP→(都道府県,市区町村,町域)。先頭への最長一致 |
| 大字・丁目・字・街区符号・住居番号 (lv4–7) | **ABR**（アドレス・ベース・レジストリ） | 町字ID/街区ID/住居IDで分解済み |
| 地番・枝番 | ABR地番 or 正規表現 | `N番地 / N番 / N号 / -N` |
| 建物・方書き (lv9–14) | 辞書 + パターン | KEN_ALL/ABR には無い |

ABR は ZIP を持たないので **KEN_ALL の ZIP で大まかに引き、その町域を ABR で引き直す**
（opa 方式）。突合は文字列で行い、団体コードの 5桁/6桁差を回避する。

**KEN_ALL の連結フィールドが ABR で割れる**（この方式の主目的の一つ）:

| KEN_ALL 市区町村名 | ABR 再解決後のラベル |
|---|---|
| `千代田区`（特別区） | `東京23区:千代田区` |
| `横浜市西区`（政令市） | `市:横浜市` ＋ `区:西区` |
| `虻田郡洞爺湖町`（郡） | `群:虻田郡` ＋ `町村:洞爺湖町` |

実装: `abr/KenAll.java`, `abr/Abr.java`, `abr/Csv.java`。列はヘッダ名参照で読むので、
実ダンプの版で列が増減しても `get("列名")` の名前だけ直せばよい。

### 3.3 データ拡張（頑健性）

実入力は表記が揺れる。`synth/Synth.augment()` が同一意味のまま表層だけ揺らす:
全角/半角数字、ハイフン異字（`- ー − － ‐`）、`丁目/番地/号 → 記号`、ZIP の `〒` 有無。
ラベルは保持されるので拡張後も正解アライメントが維持される。

## 4. 教師なし字推定（本丸）

ABR は字(小字)を網羅しない。ラベルが付くのは住所の**頭**まで。字を含む**尻尾には正解が無い**。
ここを **ラベル無しで推定**する。実装は `aza/AzaInducer.java`, `aza/Aza.java`。

### 4.1 なぜ教師なしで当たるのか（3つの根拠）

1. **残差スロット（消去法）**: ABR が `大字` を、末尾の数字塊が `地番` の開始を決める。
   → 間に残る非空スパンは、誰もラベルしなくても **構造的に `字`**。`Aza.peel()` が切り出す。
2. **反復性（教師なし字彙誘導）**: `字` 名は全国の住所で繰り返す。残差をコーパス横断で集め、
   **頻出 × 左右に多様な隣接（分岐エントロピー高）** の部分文字列を **字彙として誘導**。
3. **PMI で癒着を剪定**: 「字＋小字」が連接した残差を割る時、`中里前田` のように
   独立共起（PMI 低）な連接は合成語として **字彙から除去**。`田中` のように常に共起（PMI 高）な
   ものは1単位として保持。残った字彙でユニグラム LM の最尤分割（`segment`）。

ABR/KEN_ALL は (1) の足場（大字辞書＋数字境界）を与えるだけ。`字` のラベルは1つも要らない。

### 4.2 アルゴリズム

```
fit(residuals):
  全部分文字列(len 1..maxLen)の頻度 cnt と 左右隣接分布 left/right を集計（端=BND）
  admit: cnt>=minCount かつ 左右が多様(>=2種 or 端) かつ 平均分岐エントロピー>=eMin
  prune: 既知2分割 a+b の PMI=log(c(ab)*N/(c(a)c(b))) が tau 以下 → 合成語として削除
segment(name):
  ユニグラム logp(piece)=log(count/Z)（未知はフロア） − wordPenalty を最大化する Viterbi
```

`N`=残差件数（PMI 正規化）、`Z`=字彙頻度和（ユニグラム確率）。

### 4.3 結果（`AzaDemo`, 教師ラベル完全ゼロ）

字を隠して 400 件生成 → ラベル無し推定 → 隠した正解で採点:

```
内部境界(字の切れ目) P=1.000 R=0.708 F1=0.829
字スパン完全一致      353/400 = 0.883
  萩荘中里前田…  正解['中里','前田'] → 字小字:中里 / 字小字:前田
  真柴中里北…    正解['中里','北']   → 字小字:中里 / 字小字:北
```

精度 1.000（誤った切れ目を入れない）、再現は閾値依存（`tau`/`wordPenalty`）。
実データは数千万件あり字の反復が桁違いに効くので、玩具コーパスより遥かに頑健になる。

## 5. モデル

### 5.1 参照実装（本リポ）

`tagger/PerceptronTagger.java`: **平均化構造化パーセプトロン + Viterbi**（線形 CRF と同族・依存ゼロ）。
BIOES 遷移の合法性を Viterbi でマスクし、不正系列（I- が単独で始まる等）を排除する。
素性は `feature/Features.java`（窓 ±2 の文字・文字種・接尾辞KW・bigram）。
`tagger/AddressParser.java` が学習/parse/簡易評価の窓口。

CRF ではなくパーセプトロンにしたのは、**依存ゼロで純 JDK で動く参照実装**にするため。
精度が頭打ちになったら以下へ差し替える。

### 5.2 アップグレード経路

1. **CRF 本実装**: MALLET（`cc.mallet:mallet`）の CRF へ差し替え。素性関数はそのまま流用可。
2. **文字 BERT 微調整**（`tohoku-nlp/bert-base-japanese-char-v3` 等）+ CRF ヘッド →
   **ONNX export** → Java(`onnxruntime`) 推論。前処理は codepoint 列化のみ。
3. **ZIP を特徴/事前分布として注入**: KEN_ALL gazetteer 一致を素性化、または ZIP→lv1-4 を制約デコード。

### 5.3 教師の無い尻尾を学習する正攻法

- **self-training（半教師）**: §4 のゼロラベル推定 + §3 の頭ラベルを種にし、CRF の周辺確率で
  高信頼予測のみ擬似ラベル化 → 再学習を反復。CRF が文脈で境界を均し字彙の穴を補間。
- **partial-CRF（周辺尤度学習）**: 頭=既知ラベル、尻尾=潜在として周辺化して学習
  （`pytorch-partial-crf` の Marginal/Partial CRF が該当）。EM 的に潜在の字構造を学ぶ。
- **LLM 蒸留**: どの辞書にも無い字・通称・屋敷番は、サンプルを LLM でラベルし
  安い CRF/BERT へ蒸留して「教師を製造」。

| `tagger.PerceptronTagger` | 系列ラベラ本体（CRF 差し替え可能なスロット） |
| `tagger.AddressParser` | 学習/parse/評価の窓口 |

| `abr.Abr` | KEN_ALL→ABR 再解決 → Component 列（頭の階層分離） |

## 8. unlaxer への統合

- **CodePoint 整合**: 前処理は codepoint 列化のみ。unlaxer の CodePoint 基盤にそのまま載る。
- **ブートストラップ**: 学習済みモデルを「オラクル」として大量ラベルを生成し、unlaxer の
  パーサコンビネータ規則（地番/丁目/号 等）を自動誘導/反証する材料にする。
- **本番埋め込み**: BERT 系に上げたら ONNX 化して onnxruntime-java で推論。
- **enum 対応**: 出力ラベル文字列は `階層要素`（`label.Hierarchy`）と 1:1 対応。
- **外字/名寄せ**: codepoint 単位なので PUA がそのまま流れ、gaiji 変換テーブルと突き合わせ可能。

## 9. ロードマップ

1. **実 ABR/KEN_ALL 取込**: `kenallEncoding=CP932`、ABR 列名を実版で検証、まず住居表示の1県で回す。
2. **実評価基盤**: 手ラベル少数の hold-out、entity-level（スパン）F1、混同行列。
3. **self-training ループ**: 周辺確率の信頼度ゲートで擬似ラベルを反復追加。
4. **分岐エントロピー併用**: `AzaInducer` に明示的な境界スコアを足し PMI と併用。
5. **CRF / BERT 差し替え**: MALLET CRF or 文字BERT+ONNX。
6. **建物・方書き**: 辞書とパターンを `synth`/`augment` に追加しカバレッジ確保。
7. **partial-CRF**: 周辺尤度で尻尾を潜在として学習。

## 10. 付録: 設計判断ログ（字推定の失敗から学んだ点）

教師なし分割の閾値設計は一発では決まらなかった。経緯を残す（再発防止）:

1. **「頻度×長さ」最大化 → 過分割**（中里→中+里）。高頻度の1文字を並べた方が得をする。
2. **単語ペナルティを上げる → 癒着**（中里前田 が1単位）。真の字どうしの偶然連接が「単語」化。
3. **判別信号は PMI**: 中里と前田が独立共起なら PMI 低→分割、田中のように常に共起なら高→保持。
4. **PMI の正規化に注意**: 全部分文字列頻度和 `Z` ではなく **文字列件数 `N`** で正規化する。
   ここを誤ると PMI が一様に過大評価され剪定が効かない（実際に踏んだ）。

→ 結論: ユニグラム LM（log 確率）＋ PMI 剪定（N 正規化）＋ 控えめな単語ペナルティ、で
精度 1.000 / F1 0.83 に到達。`tau`/`wordPenalty`/`minCount` は実データの hold-out で再調整する。
